# Skill: Revisar rutas y navegación

## Objetivo

Revisar que todas las rutas del proyecto funcionen correctamente.

## Controlar

- Links del header.
- Links del footer.
- Link Admin.
- Rutas desde páginas públicas hacia admin/login.html.
- Rutas internas del dashboard.
- Links de CSS.
- Links de JavaScript.
- Links de imágenes.
- Rutas compatibles con hosting.

## Reglas

- Corregir rutas rotas.
- No duplicar archivos.
- No usar rutas absolutas innecesarias.
- Mantener compatibilidad para hosting.
- No cambiar nombres sin revisar consecuencias.

## Resultado esperado

Un sistema de navegación claro y sin enlaces rotos.
